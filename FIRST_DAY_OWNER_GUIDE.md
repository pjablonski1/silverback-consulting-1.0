# Silverback HQ First-Day Owner Guide

## What Silverback HQ Is

Silverback HQ is the operating dashboard for managing clients, appointments, tasks, onboarding, documents, invoices, assessments, and follow-up activity.

## First Login

Use the Microsoft login assigned by Azure. Aida and Michael should both have Owner/Admin access.

If testing locally, use the preview access screen only. Local preview access is not the production security model.

## First 30 Minutes

1. Open Silverback HQ.
2. Confirm the CRM is empty.
3. Add the first real client.
4. Add the main contact.
5. Add the client phase.
6. Add the next action.
7. Create the first task.
8. Schedule the first appointment.

## Daily Workflow

Use the CEO Dashboard every morning:

- Review today's priorities.
- Review overdue tasks.
- Check client follow-ups.
- Check open invoices.
- Review notification history.
- Move clients through the lifecycle pipeline.

## Client Workflow

1. Consultation booked.
2. Client record created.
3. Required documents sent.
4. Documents signed.
5. Payment requested.
6. Payment completed.
7. Client portal enabled.
8. Engagement begins.

## Business Health Assessment

Use the assessment when onboarding or reviewing a client. Complete each section, save draft as needed, and export the result for client review.

## Invoices

Create invoices from the Finance section. Enter billable hours, hourly rate, or flat fee. Payment providers should be connected before accepting live payments.

## Notification History

Use Notification History to confirm whether appointment emails were queued, sent, failed, or retried.

## What Not To Store In The CRM

Do not store:

- Passwords
- Credit card numbers
- Bank login credentials
- Private signing keys
- API keys

Use the connected provider dashboards for sensitive payment, banking, and e-signature secrets.
